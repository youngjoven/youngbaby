## 개발 순서

```
1단계: Xcode 프로젝트 + SwiftData 모델
2단계: 온보딩 화면 (Cognito 인증)
3단계: 홈 탭 (핵심 기능 - 수유·배변 기록)
4단계: 기록 탭 (통계·차트)
5단계: AWS 백엔드 (SAM + Lambda + DynamoDB)
6단계: API 연동 + 동기화
7단계: SNS/APNs 알람
8단계: 어드바이저 탭 (Bedrock)
9단계: 인사이트 탭 (Bedrock)
```

## 질의응답

**[Question] 1:** 앱 번들 ID를 결정해야 합니다. App Store 등록 시 사용되는 고유 식별자입니다.
예시: `com.youngbaby.app`, `com.{이름}.youngbaby`
**[Answer]** 네, com.youngbaby.app 이것으로 결정하겠습니다.

**[Question] 2:** 앱 표시 이름(App Display Name)을 결정해야 합니다. 홈 화면에 보이는 앱 이름입니다.
예시: `아기 수유`, `영아 수유 기록`, `밀크 노트`
**[Answer]** '아기 일기장'으로 이름을 짓습니다.

**[Question] 3:** AWS 리전(Region)을 결정해야 합니다. 한국 사용자 대상이므로 아래 중 선택합니다.
1) ap-northeast-2 (서울) — 한국 사용자 최적 응답 속도
2) us-east-1 (버지니아) — Bedrock 모델 가용성 높음
**[Answer]**  ap-northeast-2 을 사용합니다.
