## 1. 핵심 도메인 모델 (Entity)

### UserProfile
아이 및 보호자 정보를 저장하는 단일 프로필 (단일 아이 전제)

| 필드 | 타입 | 설명 |
|---|---|---|
| `id` | UUID | 고유 식별자 |
| `babyName` | String | 아이 이름 |
| `babyBirthDate` | Date | 아이 생년월일 (월령 자동 계산에 사용) |
| `motherName` | String | 어머니 이름 |
| `createdAt` | Date | 프로필 생성일 |

> **월령 계산**: `babyBirthDate`와 현재 날짜(오늘)의 차이를 월 단위로 계산. 사용자 직접 입력 불필요.
> **계산식**: `currentAgeInMonths = (today - babyBirthDate) / 30`

---

### FeedingRecord
분유 수유 1회 기록

| 필드 | 타입 | 설명 |
|---|---|---|
| `id` | UUID | 고유 식별자 |
| `feedingTime` | Date | 수유 시간 (버튼 클릭 시 현재 시각 자동 기록) |
| `amountMl` | Int | 분유량 (ml, 사용자 직접 입력) |
| `createdAt` | Date | 기록 생성일 |

---

### BowelRecord
배변 활동 1회 기록

| 필드 | 타입 | 설명 |
|---|---|---|
| `id` | UUID | 고유 식별자 |
| `bowelTime` | Date | 배변 시간 (사용자 직접 입력) |
| `bowelState` | Enum | 배변 상태: `hard`(단단함) / `normal`(정상) / `loose`(묽음) |
| `createdAt` | Date | 기록 생성일 |

> **BowelState Enum 정의**
> - `hard`: 변이 굳어 있음
> - `normal`: 너무 단단하지도 묽지도 않음
> - `loose`: 물기가 많음

## 2. 핵심 비즈니스 로직 (Domain Service)

### FeedingService
| 기능 | 설명 |
|---|---|
| `recordFeeding(amountMl)` | 현재 시각을 수유 시간으로 자동 기록, 분유량 저장 |
| `getAverageFeedingInterval()` | 최근 수유 기록들의 평균 간격 계산 |
| `getAverageAmountMl()` | 1회 평균 분유량 계산 |
| `getDailyTotalAmountMl(date)` | 특정 날짜의 하루 총 분유량 계산 |
| `getFeedingPattern(days)` | 최근 N일간 수유 패턴 분석 |
| `compareWithRecommended()` | 현재 월령 기준 권장 수유 간격·분유량과 비교 |

### BowelService
| 기능 | 설명 |
|---|---|
| `recordBowel(time, state)` | 배변 시간 및 상태 저장 |
| `getDailyBowelCount(date)` | 특정 날짜의 하루 배변 횟수 자동 집계 |
| `getBowelStateTrend(days)` | 최근 N일간 배변 상태 추이 분석 |

### FeedingBowelAnalysisService
| 기능 | 설명 |
|---|---|
| `analyzeCorrelation()` | 분유량·수유 간격과 배변 상태의 상관관계 분석 |
| `getHealthSummary()` | 수유 + 배변 종합 건강 요약 제공 |

### AgeService
| 기능 | 설명 |
|---|---|
| `getCurrentAgeInMonths()` | 생년월일 기반 현재 월령 계산 |
| `getRecommendedInterval()` | 현재 월령에 맞는 권장 수유 간격 반환 |
| `getRecommendedAmountMl()` | 현재 월령에 맞는 권장 분유량 반환 |

## 3. 알람 예측 로직

**기준:** 대한소아과학회·WHO 공개 가이드라인 (참고용, 저작권 안전 데이터만 활용)

| 월령 | 권장 수유 간격 | 하루 수유 횟수 |
|---|---|---|
| 0~1개월 | 2~3시간 | 8~12회 |
| 1~3개월 | 3~4시간 | 6~8회 |
| 3~6개월 | 3~4시간 | 5~6회 |
| 6~12개월 | 4~5시간 | 3~5회 |

**알람 예측 로직:**
1. `AgeService.getCurrentAgeInMonths()` → 현재 월령 계산
2. 현재 월령에 해당하는 권장 수유 간격 조회
3. 마지막 수유 시간 + 권장 간격 = 다음 알람 시각 자동 설정
4. 앱 내 로컬 알람(iOS UNUserNotificationCenter)으로 발송

> **면책 문구 (앱 내 표시 필수):** "권장 수유 간격 및 분유량은 대한소아과학회·WHO 가이드라인을 참고한 것으로, 의료적 진단을 대체하지 않습니다."

## 4. 스마트 수유 어드바이저 Agent 도메인 설계

**기술:** Amazon Bedrock

### 입력 (Input)
| 항목 | 설명 |
|---|---|
| 최근 수유 기록 | 최근 7일간 `FeedingRecord` 목록 |
| 최근 배변 기록 | 최근 7일간 `BowelRecord` 목록 |
| 현재 월령 | `AgeService.getCurrentAgeInMonths()` |
| 월령별 권장 기준 | 수유 간격, 권장 분유량 |

### 출력 (Output)
| 항목 | 설명 |
|---|---|
| 다음 수유 시간 조정 추천 | "다음 수유는 3시간 후를 추천합니다" |
| 분유량 조언 | "현재 월령 기준 1회 140ml를 권장합니다" |
| 수유-배변 종합 의견 | "이번 주 배변 상태가 정상 범위입니다" |

### 전달 방식
- **즉시 추천 카드**: 수유 기록 직후 화면에 자동 표시
- **어드바이저 탭**: 사용자가 직접 탭 진입 시 최신 추천 조회 가능

## 5. 데이터 인사이트 Agent 도메인 설계

**기술:** Amazon Bedrock

### 분석 트리거 조건
| 조건 | 설명 |
|---|---|
| 주간 분석 | 매주 일정 시점에 자동 실행 |
| 데이터 임계 충족 | 최소 3일 이상 기록 축적 시 인사이트 생성 가능 |

### 인사이트 유형
| 유형 | 예시 |
|---|---|
| 분유량 변화 감지 | "이번 주 분유량이 지난주 대비 15% 감소했습니다" |
| 수유 간격 변화 | "평균 수유 간격이 3시간에서 4시간으로 늘어났습니다" |
| 배변 이상 패턴 | "최근 3일간 묽음 상태가 반복되고 있습니다" |
| 월령별 권장량 비교 | "현재 월령 기준 하루 권장량(700ml)보다 적게 섭취 중입니다" |
| 수유-배변 연관 | "분유량이 많은 날 배변 횟수가 증가하는 패턴이 확인됩니다" |

### 전달 방식
- 앱 내 인사이트 카드/피드로 표시 (Push Notification 미사용)
- 앱 열었을 때 홈 화면 또는 별도 인사이트 탭에서 확인

## 6. 월령별 권장 데이터 확보 전략

- **출처**: 대한소아과학회·WHO 공개 가이드라인
- **원칙**: 저작권 보호 자료는 절대 사용하지 않음. 공개된 수치만 활용.
- **저장 방식**: 앱 내 하드코딩(Static Data)으로 관리 (0~12개월 구간별 테이블)
- **면책 문구**: 모든 권장 수치 표시 시 "참고용" 명시 필수
