## 1. 핵심 도메인 개념

### 아이 프로필 (Baby Profile)
- 앱은 **단일 아이**를 관리한다.
- 아이는 **이름**과 **생년월일**을 가진다.
- 보호자(어머니)는 **이름**을 가진다.
- **월령**은 생년월일과 오늘 날짜를 기반으로 **자동 계산**된다. 사용자가 직접 입력하지 않는다.

### 분유 수유 기록 (Feeding Record)
- 수유 1회를 하나의 기록으로 남긴다.
- **수유 시간**은 사용자가 버튼을 클릭하는 순간 자동으로 기록된다. 직접 입력하지 않는다.
- **분유량(ml)** 은 사용자가 직접 입력한다.

### 배변 기록 (Bowel Record)
- 배변 1회를 하나의 기록으로 남긴다.
- **배변 시간**은 기록 버튼 클릭 시 현재 시각으로 **자동 기록**된다. 사용자가 직접 입력하지 않는다.
- **배변 상태**는 아래 3가지 중 하나로 기록한다:
  - **단단함**: 변이 굳어 있음
  - **정상**: 너무 단단하지도 묽지도 않음
  - **묽음**: 물기가 많음
- 하루 배변 횟수는 개별 기록을 바탕으로 **자동 집계**된다.

## 2. 비즈니스 규칙

### 수유 관련 규칙
1. 수유 시간은 버튼 클릭 시 현재 시각이 자동 기록된다.
2. 분유량은 0ml 이하로 입력할 수 없다.
3. 수유 간격은 직전 수유 시간과 현재 수유 시간의 차이로 계산된다.
4. 알람 간격은 현재 월령에 따라 자동으로 결정된다.

### 배변 관련 규칙
1. 배변 시간은 기록 버튼 클릭 시 현재 시각이 자동 기록된다. 직접 입력하지 않는다.
2. 배변 상태는 반드시 단단함 / 정상 / 묽음 중 하나여야 한다.
3. 하루 배변 횟수는 당일 기록된 배변 건수를 집계한 값이다.

### 월령 관련 규칙
1. 월령은 생년월일로부터 오늘까지의 개월 수로 계산한다.
2. 이 앱의 대상 월령은 생후 0개월 ~ 12개월이다.
3. 월령이 변경되면 권장 수유 간격과 권장 분유량이 자동으로 갱신된다.

### 권장 기준 관련 규칙
1. 모든 권장 수치는 대한소아과학회·WHO 공개 가이드라인을 기반으로 한다.
2. 권장 수치는 의료적 진단을 대체하지 않으며, 앱 내에서 반드시 **"참고용"** 임을 명시한다.
3. 저작권이 있는 자료는 절대 사용하지 않는다.

## 3. 도메인 서비스

### 수유 서비스
- 버튼 클릭 시 현재 시각으로 수유 기록을 생성한다.
- 평균 수유 간격을 계산한다.
- 1회 평균 분유량을 계산한다.
- 하루 총 분유량을 계산한다.
- 수유 패턴(일정 기간 동안의 변화)을 분석한다.
- 현재 월령 기준 권장 수유 간격 및 분유량과 비교한다.

### 배변 서비스
- 배변 시간과 상태를 기록한다.
- 하루 배변 횟수를 집계한다.
- 일정 기간의 배변 상태 추이를 분석한다.

### 수유-배변 연관 분석 서비스
- 분유량·수유 간격과 배변 상태의 상관관계를 분석한다.
- 수유 + 배변 종합 건강 요약을 제공한다.

### 월령 서비스
- 생년월일로부터 현재 월령을 계산한다.
- 현재 월령에 맞는 권장 수유 간격을 반환한다.
- 현재 월령에 맞는 권장 분유량을 반환한다.

## 4. 알람 예측 로직

**기준:** 대한소아과학회·WHO 공개 가이드라인 (참고용)

| 월령 | 권장 수유 간격 | 하루 수유 횟수 |
|---|---|---|
| 0~1개월 | 2~3시간 | 8~12회 |
| 1~3개월 | 3~4시간 | 6~8회 |
| 3~6개월 | 3~4시간 | 5~6회 |
| 6~12개월 | 4~5시간 | 3~5회 |

**예측 흐름:**
1. 현재 아이의 월령을 계산한다.
2. 월령에 해당하는 권장 수유 간격을 조회한다.
3. 마지막 수유 시간 + 권장 간격 = 다음 알람 시각으로 자동 설정한다.
4. 알람은 SNS/APNs 서버 알람으로 발송된다 (엄마·아빠 다중 기기 동시 수신 지원). 세부 구현은 3.logical.md 참조.

> **면책 문구 (앱 내 표시 필수):**
> "권장 수유 간격 및 분유량은 대한소아과학회·WHO 가이드라인을 참고한 것으로, 의료적 진단을 대체하지 않습니다."

## 5. 스마트 수유 어드바이저 Agent 도메인 설계

**기술:** Amazon Bedrock
**분석 범위:** 수유 기록 + 배변 기록 통합 분석

### 분석에 사용하는 정보
- 최근 7일간 수유 기록 (수유 시간, 분유량)
- 최근 7일간 배변 기록 (배변 시간, 배변 상태)
- 현재 월령
- 월령별 권장 수유 간격 및 분유량 기준

### 제공하는 추천
- 다음 수유 시간 조정 추천 (예: "다음 수유는 3시간 후를 추천합니다")
- 분유량 조언 (예: "현재 월령 기준 1회 140ml를 권장합니다")
- 수유-배변 종합 의견 (예: "이번 주 배변 상태와 수유량을 보면 전반적으로 양호합니다")

### 전달 방식
- **즉시 추천 카드**: 수유 기록 직후 화면에 자동 표시
- **어드바이저 탭**: 사용자가 직접 탭 진입 시 최신 추천 조회 가능

## 6. 데이터 인사이트 Agent 도메인 설계

**기술:** Amazon Bedrock
**전달 방식:** 앱 내 인사이트 카드/피드 (Push Notification 미사용)

### 인사이트 생성 조건
- 최소 3일 이상 기록이 축적되어야 인사이트를 생성한다.
- 주 1회 자동으로 분석을 실행한다.

### 인사이트 유형
| 유형 | 예시 |
|---|---|
| 분유량 변화 감지 | "이번 주 분유량이 지난주 대비 15% 감소했습니다" |
| 수유 간격 변화 | "평균 수유 간격이 3시간에서 4시간으로 늘어났습니다" |
| 배변 이상 패턴 | "최근 3일간 묽음 상태가 반복되고 있습니다" |
| 월령별 권장량 비교 | "현재 월령 기준 하루 권장량(700ml)보다 적게 섭취 중입니다" |
| 수유-배변 연관 | "분유량이 많은 날 배변 횟수가 증가하는 패턴이 확인됩니다" |

## 7. 월령별 권장 분유량 기준

**출처:** 대한소아과학회·WHO 공개 가이드라인 (참고용, 저작권 안전 수치만 활용)

| 월령 | 1회 권장 분유량 | 하루 권장 총량 |
|---|---|---|
| 0~1개월 | 60~90ml | 약 500~700ml |
| 1~3개월 | 90~120ml | 약 700~900ml |
| 3~6개월 | 120~180ml | 약 800~1000ml |
| 6~12개월 | 180~240ml | 약 600~900ml (이유식 병행) |
